Known Limitations / Bugs
-----------------------

[PascalScript Studio]
- When using {$I} include directives or additional units in the uses clause,
  the debugger line numbers are no longer mapped correctly.
  As a result, incorrect lines are highlighted in the editor during step-by-step script execution.
  This is caused by included source files changing the internal line offsets used by the debugger.

[Edit Table Data]
- Case-sensitive table names are currently not supported.

[Embedded Connections]
- The user manager is hidden.
  Login is currently only possible with SYSDBA.

[Navigation Tree → Tables → Context Menu "Copy Table"]
- Computed fields are not supported.
  Example:
    first_name || ' ' || last_name AS full_name

[DB Tools → DB Info → Tab "Pages"]
- The values for "Used Pages" and "Free Pages" are swapped.
- This behavior also occurs in IBX DBAdmin.
  → Likely an IBX issue.

[RTF Export]
- The RTF file is generated.
- Opening:
  - AbiWord: works
  - LibreOffice: does not work
- Likely an outdated RTF format is being generated.
- Issue has been reported (2025-09-08).

[Data Editor]
- Currently unstable and under active development.

[Sorting with calculated columns without aliases]
- Problem: Attempting to sort a calculated column without an alias (system-generated name like CONCATENATION)
  in the grid leads to SQL error -206 (Column unknown - CONCATENATION).
  This also occurs with multi-column sorting (custom-built), because Firebird does not accept the internal
  column name for ORDER BY.

  Affected query (produces error when sorting):
  SELECT
    '1' || ' = ' || ASCII_VAL('1'),
    '2' || ' = ' || ASCII_VAL('2'),
    ...
    'Z' || ' = ' || ASCII_VAL('Z')
  FROM RDB$DATABASE;

  Workaround: Give each calculated column a unique alias, e.g. AS COL1, AS COL2, …
  After that, sorting works perfectly – both single and multi-column.

  Corrected query (works with sorting):
  SELECT
    '1' || ' = ' || ASCII_VAL('1') AS COL1,
    '2' || ' = ' || ASCII_VAL('2') AS COL2,
    ...
    'Z' || ' = ' || ASCII_VAL('Z') AS COL16
  FROM RDB$DATABASE;

  Affects: Custom-built sorting procedures in QueryWindow.
  Status: May be fixed in a future version.
