TurboBird v1.2.1.1273
Build Date & Time: 19.05.2026, 18:14:12


+ FormDesigner & PascalScript (PascalScript Studio)

- Dedicated "Forms" node in the database tree
- Custom forms can now be created per database
- Forms are stored as .cfrm + .rops files
- "New Form" opens the visual designer
- "Edit Form" loads an existing form into the designer
- "Execute Form" runs a form directly
- Double-clicking a form starts execution
- Storage location: data/PSStudio/Forms/<Server>/<Database>/

! Table Management and Constraints (completely reworked)

* All constraint types are now visible as subnodes of each table
* Primary Key with field list
* Foreign Keys with reference display
* Unique Constraints with field list
* Check Constraints with condition
* Not Null Constraints with affected fields
* Indices (user-defined only, excluding system indices)
* Multi-column keys are grouped clearly (PRIMARY KEY (Field1, Field2))
* Table management extended with new tabs:

  * Primary Key: select fields via checkbox, changes generate SQL
  * Foreign Keys: manage existing ones, create new ones with wizard
  * Unique Constraints: select fields, assign name
  * Check Constraints: edit expression directly
  * Not Null Constraints: check fields, changes generated as ALTER statements

! Trigger Support (extended)

* New trigger types in the tree:

  * Database Triggers (ON CONNECT, ON DISCONNECT, ON TRANSACTION...)
  * DDL Triggers (BEFORE/AFTER CREATE TABLE, DROP TABLE, etc.)
  * UDR Triggers (EXTERNAL NAME ... ENGINE UDR)
* "Create new Trigger" now provides the correct template depending on type
* Trigger view now displays the correct type (Table/DB/DDL)
* Editing and deleting all trigger types is now supported

! Tree Structure (improved)

* Table fields are now clearly grouped under "Fields"
* Constraints and indices are directly visible under the table
* Table triggers are directly visible under the table
* "References" and "Permissions" under each table

And several bugs fixed.

TurboBird v1.2.1.1264
Build date & time: 2026-02-16, 20:30:03

+ Activity Monitor (first version) added
  Displays Attachments, Transactions, and Statements
  Attachments can be terminated
  Statements can also be terminated (remain visible until the related attachment ends)
  Accessible via: Right-click on database node → DB-Tools → Activity Monitor
  (This feature is still under development)

+ Tree object filter added
  Simple filter for tree objects
  Container objects (e.g. Packages) remain visible to preserve structure
  Filter performance is fast and stable even with very large databases

! TreeView performance significantly improved
  Faster loading and navigation
  Tested with extremely large databases:
    10,000 Packages (each with multiple subnodes)
    1,000,000 Domains
    several thousand tables
  Tree loading and filtering remain responsive and stable

+ Multiple-file restore fully implemented
  Restore from split backup files is now supported

! ToolButton "TT" re-enabled
  Font name and font size in QueryWindow (Editor) can now be configured via this button
  Editor background color can be configured via INI file
  Editor excluded from ThemeManager to ensure stable and consistent rendering

! Internal improvements and optimizations

* Known issue:
  Server-side restore works reliably and restores large databases completely
  Client-side restore may silently produce incomplete databases
  (issue currently under investigation)
  Recommendation: use server-side restore

TurboBird v1.2.1.1263

Build Date: 2025-12-25
Build Time: 08:56:10

! Complete migration of the application from SQLDB to IBX
  (SQLDB/FPC is no longer used)

+ Support for multiple Firebird server versions within a single TurboBird instance
  (Firebird 2.5, 3.x, 4.x, 5.x, 6.0)

+ Full support for case-sensitive object names
  (tables and fields with mixed-case identifiers)
   Correct handling of quoted and unquoted identifiers
  Context menu in QueryWindow (Ctrl+Space) lists tables and fields correctly
  Case-sensitive names are automatically inserted using double quotes
  (Alias and dot-notation detection not yet implemented – currently intentionally simple)

! Script execution with and without terminators improved
  IBX correctly handles procedure bodies without SET TERM
  Terminators are still added when loading DB objects into the QueryWindow
  (for compatibility with other tools)

+ Integrated Tony Whyman’s IBX Scripter (MWA Software)

! Backup / Restore fully migrated to Tony Whyman’s IBX-based implementation
  More options available (server-side / client-side backups and restores)

! TreeView reworked
  Switched to lazy loading
  Nodes are loaded from the database only when opened for the first time
  Optional refresh behavior configurable via INI file
  Ability to show/hide the TreeView
  (provides more space for query code)

+ QueryWindow improvements
  Code view or result view can be maximized
  Navigator can be shown or hidden via INI file

! Export to Clipboard improved
  Maximum number of exported rows can be limited via INI file
  Prevents accidental copying of very large tables

! Server registration is now mandatory
  Server names are no longer derived from connection strings
  Servers must be resolvable via DNS or the hosts file

! Multi-server handling via IBX
  Loading/unloading of client libraries fully handled by IBX

- Removed custom component SQLDbExt
  (no longer required due to IBX migration)

TurboBird v1.2.1.1256
Build date & time: 08.08.2025, 10:40:06

Aug 08, 2025 – 1.2.1.1256
+ Added: Dynamic Firebird data type handling – all Firebird data types are now loaded dynamically at runtime, making TurboBird largely     version-independent.
+ Added: Support for creating and editing Firebird packages directly within TurboBird.
+ Added: UDR (User Defined Routines) support – create, edit, and execute UDRs including parameters and return values.
+ Added: Unified Routine Runner supporting UDFs, UDRs, Stored Procedures, and Functions in a single execution tool.
+ Added: “SetFireBirdClient” dialog to select Firebird client version at any time; automatically detects installed versions.
+ Added: Firebird server configuration management – import, edit, and export server settings directly from TurboBird.
+ Added: Array and DateTimeWithTimeZone support via new TSQLQueryExt component to display unsupported data types in DBGrid.
! Changed: Relocation of Turbobird.reg and Query History file to src/data/config for multi-instance configurations.
! Changed: Configuration handling now allows multiple TurboBird instances for different Firebird versions simultaneously, simplifying testing and database migration from older versions.
