Testing long-running queries

If you want to test how TurboBird handles cancelling or killing active statements, you can run a heavy query that produces millions of rows:

SELECT A.RDB$FIELD_NAME, B.RDB$FIELD_NAME
FROM RDB$FIELDS A
CROSS JOIN RDB$FIELDS B
CROSS JOIN RDB$FIELDS C;

This can generate several million rows, depending on the size of RDB$FIELDS. Use it to test the Cancel button or monitoring via MON$STATEMENTS.

Understanding MON$STATE in MON$STATEMENTS

The monitoring table MON$STATEMENTS shows the state of each active SQL statement. The field MON$STATE can have the following values:

ValueMeaning
0Idle / finished
1Active (executing)
2Fetching (cursor is open, rows being retrieved)
3Compiling / preparing statement

For long running queries you will typically see 1 or 2. There is no -1 state in Firebird; some old tools used this for "unknown".
